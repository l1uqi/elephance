Local vector memory

elephance API

A small TypeScript workspace for local LanceDB memory, agent orchestration, project schema retrieval, and Cursor integration.

View SDK View Agent View MCP Tools
@elephance/core Core SDK for memory, rule, and schema search. Published as 0.2.0.
@elephance/agent Agent orchestration for apps that own the model loop. Published as 0.2.0.
@elephance/mcp Stdio MCP server for Cursor and compatible clients. Published as 0.2.0.
LanceDB Local-first vector storage in a writable directory.

Quick start

Install and write a memory

npm install @elephance/core openai
import { configure, queryMemory, upsertMemory } from "@elephance/core";

configure({ dbPath: "./data/.lancedb" });

await upsertMemory("The user prefers concise examples.", {
  userId: "user-123",
  label: "user_preference"
});

const hits = await queryMemory("How should I answer?", { topK: 3 });

TypeScript SDK

Core API

configure(options)

Configure the database path and table names.

configure({
  dbPath: ".lancedb",
  memoryTable: "memory",
  schemaTable: "project_schema",
  ruleTable: "rule_memory"
});

upsertMemory(text, metadata)

Insert or update a short user memory.

await upsertMemory("Use pnpm in this repo.", {
  userId: "user-123",
  label: "user_preference"
});

queryMemory(query, options)

Search stored memories by semantic similarity.

const hits = await queryMemory("package manager", {
  topK: 3
});

clearUserMemory(userId)

Delete all memory rows for a user.

await clearUserMemory("user-123");

upsertRule(text, metadata)

Store a structured rule such as a project convention or coding style preference.

await upsertRule("Button radius should stay under 8px.", {
  label: "ui_preference",
  scope: "project",
  projectId: "my-app",
  action: "Keep button radius under 8px."
});

queryRules(query, options)

Search active rules and optionally record hits.

const rules = await queryRules("build a button", {
  projectId: "my-app",
  recordHit: true
});

listRules(options)

List rules for reflection, inspection, and maintenance.

replaceProjectSchemaForSource(source, updatedAt, chunks)

Replace all schema chunks for one source path.

await replaceProjectSchemaForSource(
  "tables/billing_invoice.md",
  new Date().toISOString(),
  ["## Fields\n- id\n- customer_id"]
);

queryProjectSchema(query, options)

Retrieve relevant schema chunks by meaning.

const schema = await queryProjectSchema("invoice payment join", {
  minimal: true,
  topK: 3
});

queryProjectSchemaByTableNames(tableNames, options)

Fetch schema by exact table names.

const schema = await queryProjectSchemaByTableNames([
  "billing_invoice",
  "billing_payment"
]);

batchQueryProjectSchema(keywords, options)

Run multiple schema searches and merge duplicate sources.

const schema = await batchQueryProjectSchema(
  ["invoice", "payment", "customer"],
  { mergedTopK: 4 }
);

Agent wrapper

Memory orchestration

Use the agent package when your app controls the model call. It retrieves memory and rules before the LLM request, injects compact context, extracts memory/rule candidates after the response, and writes only according to your policy. 

npm install @elephance/agent @elephance/core openai
import { createElephanceAgent } from "@elephance/agent";
import { configure } from "@elephance/core";

configure({ dbPath: "./data/.lancedb" });

const agent = createElephanceAgent({
  userId: "user-123",
  memory: {
    autoRetrieve: true,
    autoWrite: "dry-run"
  },
  rules: {
    autoRetrieve: true,
    autoExtract: true,
    autoWrite: "dry-run",
    extractor: "heuristic"
  },
  llm: {
    async chat(messages) {
      return {
        role: "assistant",
        content: "Return your model response here."
      };
    }
  }
});

const result = await agent.chat([
  { role: "user", content: "Remember that I prefer concise examples." }
]);

createElephanceAgent(options)

Create an agent wrapper around your own chat adapter.

createMemoryContext(input)

Retrieve and format relevant memory/schema context.

extractMemoryCandidates(input)

Extract durable memory candidates without writing by default.

createLlmMemoryExtractor(options)

Use your chat adapter to extract richer project conventions and UI preferences.

commitMemoryCandidates(candidates)

Write accepted candidates through the configured policy.

commitRuleCandidates(candidates)

Judge rule candidates as add, merge, conflict, or skip before writing.

selfReflectRules(options)

Dry-run rule maintenance suggestions for consolidation, conflicts, clarification, and pruning.

rules.extractor: "llm"

Self-hosted agents can opt into LLM rule extraction with the same chat adapter; MCP clients keep explicit tools.

Configuration

Cursor and environment

Run the MCP server through npx and use an absolute database path for predictable local storage. 

{
  "mcpServers": {
    "elephance": {
      "command": "npx",
      "args": ["-y", "@elephance/mcp"],
      "env": {
        "ELEPHANCE_DB_PATH": "E:\\path\\to\\your-app\\.lancedb",
        "OPENAI_API_KEY": "your-api-key",
        "OPENAI_RELAY_BASE_URL": "https://your-compatible-endpoint/v1"
      }
    }
  }
}

For local development, build the repository and run packages/mcp/dist/server.js with node. 

npm run build
node E:\github\lancedb-vector-store\packages\mcp\dist\server.js
